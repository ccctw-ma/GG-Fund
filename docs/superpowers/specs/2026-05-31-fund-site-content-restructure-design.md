# GG Fund 内容重构设计（2026-05-31）

## 背景与目标

目标是把 GG Fund 从“基金行情 + 持仓 + AI 分析”的单页工具，重构成一个更完整的中国基金与理财研究工作台。参考知乎文章《股票、基金最全网站工具推荐》的方向；由于知乎原文访问受限，本设计采用 deep-research workflow 基于公开一手来源重建能力版图，覆盖集思录、雪球、好买基金、东方财富、同花顺、上交所，以及 AKShare、Qlib、Tushare、Backtrader、Pyfolio、Streamlit 等开源/研究基础设施。

本次默认选择第一个/推荐方案，不再把设计决策交回用户。产品优先级为：投资者可用性 > 功能完备性 > 数据可解释性 > 真实交易执行。所有投资相关输出必须保留“仅供学习参考，不构成投资建议”。

## 研究结论摘要

公开工具网站能力可归纳为六类：

1. 多资产行情与数据导航：股票、基金、指数、ETF/LOF、REITs、债券、可转债、新债。
2. 基金研究工具：基金发现、筛选、横向对比、诊断评估、专题榜单。
3. 账户与转化闭环：交易入口、定投、查询、持仓、组合管理；GG Fund 本次只做模拟账户与教育路径，不做真实交易。
4. 资讯与社区内容：新闻流、7x24、视频、达人、观点、社区讨论。
5. 官方公告与高信任披露：交易所公告、基金公告、上市公司公告、监管信息核验。
6. 开放数据与量化基础设施：AKShare/AKTools 做数据接入启发，Qlib/Backtrader/Pyfolio 做研究、回测和组合分析启发。

## 推荐方案

采用“投资研究操作系统”式重构：保留现有 Next.js App Router、React 工作台、本地持仓、公开行情、DeepSeek 分析和 Supabase 登录；在前端新增内容与能力层，先通过结构化静态配置和现有接口呈现完整工具版图，再逐步接入真实数据。该方案投入可控，能够快速提升站点全貌、UI 丰富度、E2E 可测性和 README 可描述性。

不采用“只改文案和卡片”的轻量方案，因为它无法覆盖用户要求的最全能力借鉴；也不采用“立即接入 AKShare/Qlib 全量量化后端”的重型方案，因为当前项目部署在 Cloudflare Worker，Python 量化栈会显著增加部署复杂度和合规不确定性。

## 信息架构

首页 `/app` 将组织为从上到下的完整投资路径：

1. Hero：强调“基金/ETF/转债/指数/公告/AI 的一站式研究工作台”。
2. 账户总览：继续展示总资产、成本、收益、持仓、自选。
3. 市场罗盘：新增多资产入口卡片，覆盖 A 股指数、基金、ETF/LOF、REITs、债券、可转债、新债、港美/全球观察；其中非实时模块明确标注为“路线图/可接入”。
4. 工具矩阵：新增按用户任务组织的能力清单：查行情、筛基金、比基金、诊组合、看公告、读资讯、做回测、导入导出。
5. 基金工作台：保留现有市场概览、基金搜索、趋势图、加入持仓/自选、AI 分析。
6. 决策地图：扩展 BeginnerGuide，让新手理解资金期限、风险等级、市场温度、持仓权重和分批行动。
7. 研究与披露中心：新增“资讯/社区/公告/官方披露”卡片组，指向东方财富/雪球/同花顺/上交所等能力类别的抽象，不直接声称实时抓取未接入数据。
8. 开源研究底座：新增 AKShare、Qlib、Tushare、Backtrader、Pyfolio、Streamlit 启发的能力路线图卡片，解释未来可扩展的数据接入、回测、组合归因和仪表盘能力。
9. 风险提示：保留并强化免责声明。

## 组件与数据边界

新增配置型模块，避免把所有内容塞进 `App.tsx`：

- `frontend/src/researchCatalog.ts`：定义资产导航、工具矩阵、研究源、开源能力、推荐行动路径等静态配置与类型。
- `frontend/src/components/ToolUniverse.tsx`：展示多资产行情/工具矩阵/研究源/开源底座。
- `frontend/src/components/DecisionCenter.tsx`（可选命名）：若 BeginnerGuide 扩展过大，则拆出进阶决策模块；否则优先在现有 BeginnerGuide 内做小步增强。
- `frontend/src/App.tsx`：只负责状态编排、把 summary、indices、selectedFund 传给组件。

现有 API 保持不破坏：`/api/market/indices`、`/api/funds/*`、`/api/ai/analyze-fund`、本地存储与 Supabase 登录继续工作。新增内容初期不调用外部实时接口，所有未接入能力用“路线图/待接入/可扩展”标记，防止误导用户。

## UI 方向

继续沿用当前深色玻璃拟态、金色 CTA、mint/blue 状态色和卡片化布局，但加强“专业金融终端”的信息密度：

- 卡片层级：一级模块使用大标题 + Kicker；二级能力用图标、标签、状态胶囊和短说明。
- 视觉结构：大模块采用 2-4 列响应式网格，移动端自动单列。
- 状态表达：已接入、可接入、路线图三种状态明确区分。
- 风险表达：所有交易、诊断、AI、回测相关模块都保留非投资建议提示。
- 文案风格：面向基金新手也能理解，同时给进阶用户看到 ETF/LOF/REITs/转债/回测/公告等关键词。

## 错误处理与合规

现有接口错误展示方式保持不变。新增静态能力模块无需网络错误处理，但必须避免暗示真实交易或保证收益。真实数据源未接入时不能写成“实时提供”，而写成“能力路线图”“可接入来源”“当前以教育说明呈现”。

DeepSeek 分析继续走服务端，缺少 API Key 时使用本地确定性报告。AI 结论必须保留风险等级、观察点、情景推演和免责声明。

## 测试策略

本次实现需补齐以下验证：

1. 单元/组件测试：验证新增能力目录渲染、多资产导航、研究源、开源底座、状态标签和免责声明。
2. App 测试：验证首页文案更新、工具矩阵存在、旧能力仍存在。
3. E2E：扩展 Playwright 流程，覆盖根路径跳转、工具矩阵可见、基金搜索、选择基金、加入持仓、AI 分析、导入导出、错误提示。
4. 质量门禁：按 `AGENTS.md` 运行 `bun run lint`、`bun run test`、`bun run coverage`、`bun run build`、`bun run test:e2e`；Midscene 行为若涉及更新，则运行 `bun run test:midscene`。
